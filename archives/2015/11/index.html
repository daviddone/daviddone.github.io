Step 6:将接收到的消息展现到消息中心

个推的客户端消息接收机制是BroadcastReceiver，这里在广播接受的onReceive方法中获取到推送收到的详情。

收到消息之后，将数据存入到数据库中，然后标记为未读状态，等到点击之后，标记为已读状态。

问题

最好将推送机制放在服务器端，如果App分为用户端和商户端，根据Android系统机制，包名需要不同。包名不同就会导致绑定的三方推送平台AppKey等不同。

ClientId没有生成？

没有联网的情况，ClientId是无法生成的。

ClientId安装一次就会生成一次，需要绑定别名才能使用？
是的。

能够收到通知，但是透传消息无法推送？

Android端的清单文件中的AppId,AppKey一定不要填错了，注意AppId将会填写多次，在清单文件中receiver标签appId写错之后，透传消息不能够推送到Android应用中。

Android SDK 镜像更新

当遇到代理不稳定的情况，可以采用国内镜像资源更新Android SDK。不得不说，有时候镜像资源的速度还是非常给力的。这里推荐一下：国内镜像资源。

Step 1

打开Android SDK Manager 选择 Tools –> Options,然后在 Proxy Settings的Http Proxy Server和Http Proxy Port分别填上mirrors.neusoft.edu.cn和80，勾选上 [Force https://… sources to be fetched using http]，然后点击Close。

Step 2

然后选择Android SDK Manager下的Packages和Reload，这时候就可以更新了。
