Hadoop集群搭建hadoop2.X版本

hadoop集群简介

hadoop集群是多台hadoop机器进行分布式架构。hadoop伪分布式指的是在单机进行架构。测试环境，可以采用VMware虚拟机 –>管理–> 完整克隆的方式，搭建几台centos服务器，并在每台机器上安装好hadoop单机节点，针对每台机器进行hadoop安装，可以选择逐一安装，也可以采用scp命令进行拷贝。

hadoop集群搭建

1 每个节点修改hosts文件 vim /etc/hosts

192.168.1.10    master   #  主机 
192.168.1.170   david    # 工作机 slave

退出登录，重新进来，测试：ping david # master上操作

2 Master访问所有slave节点配置免密访问

ssh-keygen -t rsa -P ‘’ # master操作，一直回车即可
就将master的公钥给所有的slave服务器，slave服务器将其公钥写入授权。
scp -r /root/.ssh/id_rsa.pub root@192.168.1.10:/home/ # 远程拷贝到slave服务器 home目录下
cat /home/id_rsa.pub >> /root/.ssh/authorized_keys # slave服务器写入授权
chmod 600 /root/.ssh/authorized_keys # ll 查看权限，不足的时候赋予权限

3 配置slaves文件

vim slaves # master主机 目录 /opt/hadoop/etc/hadoop/
localhost # 保留master主机的localhost，使其同时作为datanode。
david
注意：工作机david需要修改工作机相关信息：1 core-site.xml 中的dfs主机，目录为/opt/hadoop/etc/hadoop/ 2 删掉data临时数据：/home/hadoop/tmp，删该文件夹的所有文件 rm -rf /home/hadoop/tmp 3 yarn-site.xml的主机地址 4 将hdfs-site.xml，dfs.replication可设置为1，datanode每个节点只能存放一个副本 . 5 所有修改完毕之后需要hdfs namenode -format,目录 /opt/hadoop/bin

4 启动集群

start-dfs.sh # 目录/opt/hadoop/sbin 会提示启动工作机davd
start-yarn.sh
mr-jobhistory-daemon.sh start historyserver # 开启MapReduce JobHistory Server
启动master节点即可，slave节点在master节点启动之后会自动启动。

5 关闭集群

stop-yarn.sh
stop-dfs.sh
mr-jobhistory-daemon.sh stop historyserver

jps验证集群　
master节点服务：
6606 NameNode
7236 Jps
6779 SecondaryNameNode
6941 ResourceManager
3636 JobHistoryServer

slave节点的服务：
4086 Jps
3812 DataNode
3927 NodeManager
注意：如果出现could only be replicated to 0 nodes instead of minReplication (=1). There are 0 datanode(s) running and no node(s) ，这种情况是集群没能够搭建成功。删除掉hdfs工作数据文件，进行格式化之后hdfs namenode -format，重新启动，检查master节点是否有datanode，必须要有datanode。

集群的hdfs的使用

hdfs dfs -put /home/a.log hdfs://master:9000/ #将文件a.log 上传到hdfs上去。hdfs dfs命令也可以采用hadoop fs进行替换，在hadoop1.x中使用hadoop fs命令进行操作
hdfs dfs -ls / #查看hdfs的所有目录, 或 hadoop fs -ls hdfs://master:9000/
hdfs dfs -mkdir -p /user/hadoop/david/test #创建文件
hdfs dfs -mkdir -p /user/hadoop

可通过web ui查看: http://192.168.1.10:50070 # 需要关闭防火墙 service firewalld stop

hadoop2.6.0版本官网集群：http://hadoop.apache.org/docs/r2.6.0/hadoop-project-dist/hadoop-common/ClusterSetup.html

Hadoop单机安装教程 hadoop2.X版本

修改主机名和IP的映射关系（单机可选）

修改主机名

hostname master #centos 6.x
hostnamectl set-hostname “david” # centos 7

修改hosts配置文件，修改对应主机名称

vim /etc/hosts
192.168.31.166 master （必须） # echo $HOSTNAME 查看主机名称

关闭防火墙 centos 6.4

service iptables status #查看防火墙状态
service iptables stop #关闭防火墙
chkconfig iptables –list #查看防火墙开机启动状态
chkconfig iptables off #查看防火墙开机启动状态

关闭防火墙 centos 7

systemctl start firewalld #启动firewall
systemctl stop firewalld #停止firewall
systemctl status firewalld # 查看防火墙状态

安装jdk 并设置环境变量( 参考《Linux安装Java》)

注意java安装之后所在的文件夹，以及java_home

下载安装hadoop2.6.0

伪分布式需要修改5个配置文件

1配置hadoop /opt/hadoop/etc/hadoop 文件下

第一个：hadoop-env.sh

vim hadoop-env.sh

注意：查看环境变量：echo $JAVA_HOME

第二个：core-site.xml

<!-- 指定HADOOP所使用的文件系统schema（URI），HDFS的老大（NameNode）的地址 -->
<property>
    <name>fs.defaultFS</name>
    <value>hdfs://192.168.31.166:9000</value>
</property>
<!-- 指定hadoop运行时产生文件的存储目录 -->
<property>
    <name>hadoop.tmp.dir</name>
    <value>/home/hadoop/tmp</value>
</property>

第三个：hdfs-site.xml

<!-- 指定HDFS副本的数量 -->
<property>
    <name>dfs.replication</name>
    <value>1</value>
</property>

第四个：mapred-site.xml

mv mapred-site.xml.template mapred-site.xml # 首先合并
vim mapred-site.xml

<!-- 指定mr运行在yarn上 -->
<property>
    <name>mapreduce.framework.name</name>
    <value>yarn</value>
</property>

第五个：yarn-site.xml

注意：第一个配置参数hostname，需要配置192.168.1.10服务器 ，master

hadoop添加到环境变量

vim /etc/profile

配置之后使其立马生效
source /etc/profile

格式化namenode 初始化

hdfs namenode -format    #目录 /opt/hadoop/bin

注意：如果出现配置文件配置有误，这一步无法通过。

启动hadoop

sbin/start-dfs.sh    #先启动HDFS 目录/opt/hadoop/
sbin/start-yarn.sh # 再启动YARN
( 全部启动   sbin/start-all.sh)

12095 NodeManager
12000 ResourceManager
6144 DataNode
12125 Jps
6392 SecondaryNameNode
6042 NameNode

验证是否启动成功

使用jps命令验证
27408 NameNode

27643 SecondaryNameNode

27512 DataNode

http://192.168.31.166:50070 （HDFS管理界面）
http://192.168.31.166:8088 （MR管理界面）

上传文件到hdfs上

hdfs dfs -put slaves.sh hdfs://192.168.31.166:9000/

免密登录：

ssh-keygen -t rsa -P ‘’ 一直回车即可。会生成公钥，私钥。公钥是给目标服务器的，例如A服务器想访问B服务器。就将A的公钥给B服务器，B服务器将其公钥并写入授权。相当于B把A给他的门安上了,A还留着独一无二的钥匙。

自身免密登录

cat /root/.ssh/id_rsa.pub >> /root/.ssh/authorized_keys # 将自己的公钥进行授权

A服务器免密访问B服务器

scp -r /root/.ssh/id_rsa.pub root@192.168.1.10:/home/ # 远程拷贝到B服务器 home目录下
cat /home/id_rsa.pub >> /root/.ssh/authorized_keys # 写入授权
chmod 600 /root/.ssh/authorized_keys # ll 查看权限，不足的时候赋予权限

hadoop镜像地址：http://www.apache.org/dyn/closer.cgi/hadoop/common/
ssh免密登录资料：http://chenlb.iteye.com/blog/211809
