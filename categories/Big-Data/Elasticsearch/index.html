Elasticsearch简介

ElasticSearch是一个基于Lucene的搜索服务器。它提供了一个分布式多用户能力的全文搜索引擎，基于RESTful web接口。Elasticsearch是用Java开发的，并作为Apache许可条款下的开放源码发布，是当前流行的企业级搜索引擎。设计用于云计算中，能够达到实时搜索，稳定，可靠，快速，安装使用方便。
应用场景：github，wiki。携程的关键词搜索，58同城的搜索，海量数据搜索，并且要求毫秒级别的响应。

ElasticSearch安装

1 ElasticSearch解压

tar zxvf elasticsearch-2.2.0.tar.gz

2 ElasticSearch修改配置

修改config/elasticsearch.yml (注意要顶格写，冒号后面要加一个空格)

a)cluster.name: chenkl   (同一集群要一样)
b)node.name： david  (同一集群要不一样)
c)network.Host: david   # 这里不能写127.0.0.1,否则只能本机访问。这里配域名 ip均可

3 创建ec用户

由于ElasticSearch不能采用root启动，所以必须用非root账户启动。这里创建ec用户。

useradd  ec  # 创建ec用户，默认存在同名组下，即ec组
passwd ec  # 修改ec用户的密码
sudo  chown -R ec  elasticsearch-2.2.0  # 修改文件夹权限

4 启动

./bin/elasticsearch -d    # 后台启动

5 验证ElasticSearch

curl http://david:9200/


注意：关掉防火墙

关闭elasticsearch

ps -ef | grep elastic # 找到进程


kill -9 3134   ＃ 杀掉进程

ElasticSearch集群配置

1 分发es包

scp -r elasticsearch-2.2.0 root@david2:/home/david

2 修改config/elasticsearch.yml

a)cluster.name: chenkl (同一集群要一样 保持原样)
b)node.name： david2 (同一集群要不一样)
c)network.Host: david2 # 这里不能写127.0.0.1,否则只能本机访问。这里配域名 ip均可

问题

Q：jvm内存不足 出现的错误

OpenJDK 64-Bit Server VM warning: INFO: os::commit_memory(0x000000077ad30000, 2060255232, 0) failed; error='Cannot allocate memory' (errno=12)
# There is insufficient memory for the Java Runtime Environment to continue.
# /tmp/jvm-12496/hs_error.log

A : 配置jvm的内存
es 2.2 可修改ES_HOME/bin/service/elasticsearch.conf，set.default.ES_HEAP_SIZE=1024，该值根据机器的配置可自定义。
