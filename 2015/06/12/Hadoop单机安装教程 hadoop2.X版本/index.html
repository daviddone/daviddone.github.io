Hadoop单机安装教程 hadoop2.X版本

修改主机名和IP的映射关系（单机可选）

修改主机名

hostname master #centos 6.x
hostnamectl set-hostname “david” # centos 7

修改hosts配置文件，修改对应主机名称

vim /etc/hosts
192.168.31.166 master （必须） # echo $HOSTNAME 查看主机名称

关闭防火墙 centos 6.4

service iptables status #查看防火墙状态
service iptables stop #关闭防火墙
chkconfig iptables –list #查看防火墙开机启动状态
chkconfig iptables off #查看防火墙开机启动状态

关闭防火墙 centos 7

systemctl start firewalld #启动firewall
systemctl stop firewalld #停止firewall
systemctl status firewalld # 查看防火墙状态

安装jdk 并设置环境变量( 参考《Linux安装Java》)

注意java安装之后所在的文件夹，以及java_home

下载安装hadoop2.6.0

伪分布式需要修改5个配置文件

1配置hadoop /opt/hadoop/etc/hadoop 文件下
第一个：hadoop-env.sh
vim hadoop-env.sh

注意：查看环境变量：echo $JAVA_HOME
第二个：core-site.xml


dfs.replication
1


mapreduce.framework.name
yarn


yarn.resourcemanager.hostname
master


yarn.nodemanager.aux-services
mapreduce_shuffle

export JAVA_HOME=/usr/lib/jvm/java-1.7.0-openjdk  
export HADOOP_HOME=/opt/hadoop
export PATH=$PATH:$JAVA_HOME/bin:$HADOOP_HOME/bin:$HADOOP_HOME/sbin

```
配置之后使其立马生效
source /etc/profile

格式化namenode 初始化

hdfs namenode -format    #目录 /opt/hadoop/bin

注意：如果出现配置文件配置有误，这一步无法通过。

启动hadoop

先启动HDFS
sbin/start-dfs.sh    #目录/opt/hadoop/
再启动YARN
sbin/start-yarn.sh

( 全部启动 sbin/start-all.sh)

hdfs dfs -put slaves.sh hdfs://192.168.31.166:9000/

免密登录：

ssh-keygen -t rsa -P ‘’ 一直回车即可。会生成公钥，私钥。公钥是给目标服务器的，例如A服务器想访问B服务器。就将A的公钥给B服务器，B服务器将其公钥并写入授权。相当于B把A给他的门安上了,A还留着独一无二的钥匙。

自身免密登录

cat /root/.ssh/id_rsa.pub >> /root/.ssh/authorized_keys # 将自己的公钥进行授权

A服务器免密访问B服务器

scp -r /root/.ssh/id_rsa.pub root@192.168.1.10:/home/ # 远程拷贝到B服务器 home目录下
cat /home/id_rsa.pub >> /root/.ssh/authorized_keys # 写入授权
chmod 600 /root/.ssh/authorized_keys # ll 查看权限，不足的时候赋予权限
