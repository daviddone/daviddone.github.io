Linux►crontab

Ubuntu使用crontab执行定时任务

安装crontab

apt-get install cron

crontab服务

service cron status # 查看状态
service cron start #启动
service cron restart #重启

按照语法要求配置crontab任务

crontab 语法规范

{minute} {hour} {day-of-month} {month} {day-of-week} {full-path-to-shell-script}

案例：
每个工作日23:59都进行备份作业

59 23 * * 1,2,3,4,5  shell /home/backup.sh

每一分钟执行一次python脚本：

*/1 * * * * python /home/test.py

配置crontab任务

方案一 当前用户配置定时任务

a进入编辑环境: crontab -e

*/1  *  *  *  *  echo "aaaa" >> /home/test.txt 
*/59 * * * * python /opt/test.py >/dev/null 2>&1   # 59分钟执行一次python脚本，  >/dev/null 2>&1代表不向控制台输出数据

b 查看文件是否写入数据

tail -f /home/test.txt

方案二 为指定用户配置定时任务

设置配置文件：
vim /etc/crontab

*/1  *  *  *  * root echo "aaaa" >> /home/test.txt   # 每分钟执行一次，注意有用户存在

任务不执行，查看日志：

开启日志：
1 修改rsyslog

vim /etc/rsyslog.d/50-default.conf 
cron.*              /var/log/cron.log #将cron前面的注释符去掉

2 重启rsyslog

service rsyslog  restart

3 查看日志

cat /var/log/cron.log 
tail -n 100 /var/log/cron.log    #查看日志文件最新的100行

其余重要操作：

查看命令手册：man 5 crontab
编辑Cron作业
crontab -e
对Cron作业进行列表
crontab -l
移除Cron作业
crontab -r

问题：No MTA installed, discarding output
解决：去掉所有print打印到控制台的语句，或者使用>/dev/null 2>&1代表不向控制台输出数据。

Zookeeper集群搭建教程

Zookeeper简介:

其英文意思是：动物园管理员，hadoop生态中的很多分布式应用都是小动物。zookeeper是用来集群管理，配置管理，锁服务等分布式协调服务的。分布式服务难以管理。我理解的zookeeper就是一个协调管理分布式节点的。比如，9台zookeeper集群可以管理1000台节点。zookeeper经常用来做Hbase的高可用，而zookeeper本身就是高可用的实力代表。zookeeper范围非常广， 只要想要保证集群 一致性和高可用 都需要使用到zookeeper，从hadoop到dubble。zookeeper集群通过内部选取方法确保zookeeper集群中始终都存在着Leader进行高可用。

Zookeeper安装

1 下载解压

wget http://apache.fayea.com/zookeeper/zookeeper-3.3.6/zookeeper-3.3.6.tar.gz
tar zxvf zookeeper-3.3.6.tar.gz

2 设置环境变量（可选）

3 修改配置文件 zoo.cfg

初次使用ZooKeeper，需要将/home/david/zookeeper-3.3.6/conf目录下的zoo_sample.cfg文件重命名为zoo.cfg。 重命名代码：mv zoo_sample.cfg zoo.cfg
vim zoo.cfg,按照如下代码配置 

tickTime=2000
initLimit=10
syncLimit=5
dataDir=/home/data                        # 存放工作数据的临时的，更改为/home/data
clientPort=2181
server.0=192.168.1.10:2888:3888       # 此为单机配置方案，集群配置参考如下

4 启动服务

sh zkServer.sh start # /home/david/zookeeper-3.3.6/bin目录下
sh zkServer.sh stop # 停止服务
sh zkServer.sh status # 查看状态 启动成功的标志是：出现启动状态Mode: leader 或者follower
问题： 查看状态 之后，发现Error contacting service. It is probably not running. 这个时候在工作目录下建立myid文件，并写入状态码。

5 配置集群

vim zoo.cfg #目录：home/david/zookeeper-3.3.6/conf

server.1=192.168.1.170:2888:3888 
server.2=192.168.1.159:2888:3888 
server.3=192.168.1.10:2888:3888

工作数据存放 /home/data 文件夹下建立文件myid，不同服务器分别设置1 2 3　echo 1 >> myid
启动集群，分别启动各个节点：sh zkServer.sh start # 分别启动每台机器　需要启动三台
问题：配置集群之后，注意将防火墙关闭。如果遇到启动不成功的情况，查看日志/home/david/zookeeper-3.3.6/bin/zookeeper.out，然后进行排查即可。

6 客户端连接zookeeper

sh zkClient.sh # 服务器上保存1M以下的配置文件

zookeeper 镜像资源下载路径：http://www.apache.org/dyn/closer.cgi/zookeeper/

Hadoop集群搭建hadoop2.X版本

hadoop集群简介

hadoop集群是多台hadoop机器进行分布式架构。hadoop伪分布式指的是在单机进行架构。测试环境，可以采用VMware虚拟机 –>管理–> 完整克隆的方式，搭建几台centos服务器，并在每台机器上安装好hadoop单机节点，针对每台机器进行hadoop安装，可以选择逐一安装，也可以采用scp命令进行拷贝。

hadoop集群搭建

1 每个节点修改hosts文件 vim /etc/hosts

192.168.1.10    master   #  主机 
192.168.1.170   david    # 工作机 slave

退出登录，重新进来，测试：ping david # master上操作

2 Master访问所有slave节点配置免密访问

ssh-keygen -t rsa -P ‘’ # master操作，一直回车即可
就将master的公钥给所有的slave服务器，slave服务器将其公钥写入授权。
scp -r /root/.ssh/id_rsa.pub root@192.168.1.10:/home/ # 远程拷贝到slave服务器 home目录下
cat /home/id_rsa.pub >> /root/.ssh/authorized_keys # slave服务器写入授权
chmod 600 /root/.ssh/authorized_keys # ll 查看权限，不足的时候赋予权限

3 配置slaves文件

vim slaves # master主机 目录 /opt/hadoop/etc/hadoop/
localhost # 保留master主机的localhost，使其同时作为datanode。
david
注意：工作机david需要修改工作机相关信息：1 core-site.xml 中的dfs主机，目录为/opt/hadoop/etc/hadoop/ 2 删掉data临时数据：/home/hadoop/tmp，删该文件夹的所有文件 rm -rf /home/hadoop/tmp 3 yarn-site.xml的主机地址 4 将hdfs-site.xml，dfs.replication可设置为1，datanode每个节点只能存放一个副本 . 5 所有修改完毕之后需要hdfs namenode -format,目录 /opt/hadoop/bin

4 启动集群

start-dfs.sh # 目录/opt/hadoop/sbin 会提示启动工作机davd
start-yarn.sh
mr-jobhistory-daemon.sh start historyserver # 开启MapReduce JobHistory Server
启动master节点即可，slave节点在master节点启动之后会自动启动。

5 关闭集群

stop-yarn.sh
stop-dfs.sh
mr-jobhistory-daemon.sh stop historyserver

6 jps验证集群　

master节点服务：
6606 NameNode
7236 Jps
6779 SecondaryNameNode
6941 ResourceManager
3636 JobHistoryServer

slave节点的服务：
4086 Jps
3812 DataNode
3927 NodeManager
注意：如果出现could only be replicated to 0 nodes instead of minReplication (=1). There are 0 datanode(s) running and no node(s) ，这种情况是集群没能够搭建成功。删除掉hdfs工作数据文件，进行格式化之后hdfs namenode -format，重新启动，检查master节点是否有datanode，必须要有datanode。

集群的hdfs的使用

hdfs dfs -put /home/a.log hdfs://master:9000/ #将文件a.log 上传到hdfs上去。hdfs dfs命令也可以采用hadoop fs进行替换，在hadoop1.x中使用hadoop fs命令进行操作
hdfs dfs -ls / #查看hdfs的所有目录, 或 hadoop fs -ls hdfs://master:9000/
hdfs dfs -mkdir -p /user/hadoop/david/test #创建文件
hdfs dfs -mkdir -p /user/hadoop

可通过web ui查看: http://192.168.1.10:50070 # 需要关闭防火墙 service firewalld stop

hadoop2.6.0版本官网集群：http://hadoop.apache.org/docs/r2.6.0/hadoop-project-dist/hadoop-common/ClusterSetup.html

Centos 6.x 安装Redis 源码编译

前提条件

对于Centos 6.x可以采用源代码编译安装centos，而为了要编译就需要安装 Make 和 GCC。
如果没有安装过 GCC 和 Make，使用 yum 安装。
yum install -y gcc make
如果使用Centos 7,可采用 yum install redis 直接安装。

官网下载redis源码

curl http://download.redis.io/releases/redis-3.0.4.tar.gz -o redis-3.0.4.tar.gz #解压
tar zxvf redis-3.0.4.tar.gz
mv redis-3.0.4 /usr/local/redis
cd /usr/local/redis
make && make install #编译安装

设置配置文件路径

mkdir -p /etc/redis #创建配置文件路径
cp redis.conf /etc/redis/
vi /etc/redis/redis.conf #修改配置文件

daemonize yes （no-->yes）    #默认以后台程序方式运行
# bind 127.0.0.1   # 注释掉绑定本地，支持远程访问。或可以绑定指定访问ip白名单。
requirepass foobared # 将其注释去掉,foobared即为密码

配置密码（可选，远程连接和集群建议）：requirepass foobared # 将其注释去掉,foobared即为密码
或在客户端命令行中，设置密码：config set requirepass foobared 查看密码：config get requirepass

启动redis

/usr/local/bin/redis-server /etc/redis/redis.conf
重启：service redis restart # 在源码安装方式下无效
查看启动
ps -ef | grep redis # 其运行的端口号为6379

##使用redis-cli客户端
redis-cli # 启动命令
设置密码之后使用密码登录：
redis-cli -h 127.0.0.1 -p 6379 -a foobared 或者：redis-cli -a foobared 

>set name david
OK
>get name

查看数据条数
dbsize
查看redis版本号

> info

redis_version:2.8.19
redis_git_sha1:00000000

退出客户端
exit
通过客户端关闭redis
redis-cli shutdown
redis-cli -h 127.0.0.1 -p 6379 shutdown # 查看redis启动状态，已经变为关闭状态

redis开机自启动配置

echo “/usr/local/bin/redis-server /etc/redis/redis.conf &” >> /etc/rc.local
注意：开机启动要配置在 rc.local 中，而 /etc/profile 文件，要有用户登录了，才会被执行。

防火墙开启6379远程连接端口

/etc/init.d/iptables status
如果没有开启，添加端口号配置
vi /etc/sysconfig/iptables
添加代码：

重启防火墙使端口更改生效
service iptables restart
或者直接关闭防火墙： service iptables stop

注意：远程无法访问redis，主要原因就是redis.conf配置文件中绑定了ip或者防火墙端口号的问题。

dubbo搭建教程

dubbo简介

dubbo是一个分布式服务框架，致力于提供高性能和透明化的RPC远程服务调用方案，是阿里巴巴SOA服务化治理方案的核心框架，每天为2,000+个服务提供3,000,000,000+次访问量支持，并被广泛应用于阿里巴巴集团的各成员站点。

dubbo解决的问题：个人理解，远程的方法能够本地调用，生产者提供服务接口，并实现其业务方法，消费者远程调用同名的接口实现。比webservice更加的强大。

dubbo拓展：当当网在dubbo的基础上，开源了dubbox, 主要增加了REST API。让浏览器就具备了消费者的功能。

dubbo的两种使用方法
方法一：Dubbo采用全Spring配置方式，对应用没有任何API侵入，只需用Spring加载Dubbo的配置即可，Dubbo基于Spring的Schema扩展进行加载。
方法二：通过API进行调用（不推荐）
